# Prometheus Agent & GitHub Tools — Implementation Plan

> **Project:** Themis CLI (Multi-Agent Coding Assistant)
> **Goal:** Add Prometheus, a GitHub/version-control specialist agent, with full git workflow tools and GitHub OAuth 2.0 login.

---

## Overview

The Prometheus agent sits alongside the existing Zeus, Athena, Hephaestus, Apollo, Hermes, and Ares agents in the Themis CLI. It is the single source of truth for all git and GitHub operations.

```
User Request
     │
     ▼
  Zeus 🗲  (orchestrator)
     │ delegates git/PR tasks
     ▼
Prometheus 🔥  (git specialist)
     │ calls tools
     ▼
┌──────────────────────────────────────────────┐
│  github_status → github_login  (OAuth 2.0)   │
│  git_clone, git_checkout_new_branch          │
│  git_diff, git_status, git_add               │
│  git_commit, git_push (with retry)           │
│  git_create_pr (with Themis signature)       │
└──────────────────────────────────────────────┘
```

## Step 2 — GitHub OAuth 2.0 (`internal/auth/`)

**File:** `internal/auth/github_oauth.go`

Implements the **GitHub Device Authorization Flow** — the correct approach for CLI apps (no redirect server).

### Flow

```
Themis  ──POST──▶  github.com/login/device/code
         ◀── device_code, user_code, verification_uri

Themis shows user:
  "Open https://github.com/login/device  →  enter code XXXX-XXXX"

Themis  ──POST──▶  github.com/login/oauth/access_token  (polls every 5s)
         ◀── access_token  (once user clicks Authorize)

Themis stores token at:  ~/.config/themis/github_token.json  (0600 perms)
Themis sets env:  GH_TOKEN, GITHUB_TOKEN  (so gh CLI + git HTTPS work)
```

### Key Functions

| Function | Purpose |
|---|---|
| `RequestDeviceCode(ctx)` | Requests device code from GitHub |
| `PollForToken(ctx, deviceCode, interval)` | Polls GitHub; handles `authorization_pending`, `slow_down`, `expired_token`, `access_denied` |
| `Login(ctx)` | Full flow: returns display instructions + async channel for result |
| `LoadToken()` | Reads `~/.config/themis/github_token.json` |
| `SaveToken(token)` | Writes token file with 0600 permissions |
| `IsLoggedIn()` | Returns `true` if a valid stored token exists |
| `GetToken()` | Returns the raw access token string |
| `Logout()` | Deletes the token file |

### Token File Format
```json
{
  "access_token": "gho_...",
  "token_type": "bearer",
  "scope": "repo",
  "created_at": "2026-04-19T00:00:00Z",
  "username": "octocat"
}
```

## Step 3 — New Tool Handlers (`internal/tools/github_tools.go`)

**File:** `internal/tools/github_tools.go`

Each handler follows the standard `func(ctx Context) models.ToolResponse` signature.

| Handler | Tool Name | Auth Required | Description |
|---|---|---|---|
| `HandleGitHubLogin` | `github_login` | No | Starts OAuth device flow, blocks until user completes auth |
| `HandleGitHubLogout` | `github_logout` | No | Removes stored token |
| `HandleGitHubStatus` | `github_status` | No | Returns logged-in user or "not authenticated" |
| `HandleGitAdd` | `git_add` | No | Stages files; defaults to `-A` (all) |
| `HandleGitPush` | `git_push` | **Yes** | Pre-checks status → pushes via `gitx.Push` |
| `HandleGitCreatePR` | `git_create_pr` | **Yes** | Injects diff summary → calls `gitx.CreatePR` |
| `HandleGitCheckoutNewBranch` | `git_checkout_new_branch` | No | Creates and switches to new branch |

> **Auth guard:** Both `git_push` and `git_create_pr` call `requireAuth()` at the top and return a descriptive error if `auth.IsLoggedIn()` is false.

## Step 5 — Define Prometheus Agent

### 5a. Prompt (`internal/prompt/agents/07_prometheus.go`)

Filename convention: `07_` prefix, alphabetically after Ares (`06_`).

Key prompt directives:
1. Never commit to `main`/`master`
2. Always follow the ordered workflow (see below)
3. Enforce authentication before push — check `github_status`, call `github_login` if needed
4. Check `git_diff` before creating PR
5. Check `git_status` before push (abort if dirty working tree)
6. Use conventional commit messages (`feat:`, `fix:`, `refactor:`, `docs:`, `chore:`)
7. Error recovery: retry on network errors, report merge conflicts, don't force-push

**Enforced Workflow Order:**
```
a. git_clone          (new repo only)
b. github_status      (auth check)
c. github_login       (if not authenticated)
d. git_checkout_new_branch
e. [code changes by Hephaestus/user]
f. git_diff           (review changes)
g. git_status         (verify staging state)
h. git_add
i. git_commit
j. git_status         (verify clean before push)
k. git_push
l. git_create_pr
```
